## [LIMITATION]
1. Tokenizer is space-based — Korean morphological analysis (e.g., konlpy/mecab) would reveal finer-grained patterns (postpositions, verb stems).
2. "Novel vocabulary" in test/dev may be underestimated because quoted brand names are tokenized as single tokens.
3. Text-only answerability analysis uses token-level priors; phrase-level or question-level classifiers may find stronger signals.
4. Dev agreement analysis uses majority-vote slot as ground truth — cases with 2/5 max agreement have no reliable ground truth.
5. Answer category classification used keyword matching — multi-token answers may be miscategorized (e.g., '종이컵 홀더' is classified as 재질 when 홀더 is the key noun).

## Key Actionable Insights for Model Development

1. **Stable template structure**: All 13 templates appear in both train and test with <2% distribution shift — template-aware prompting or fine-tuning is safe.
2. **Recycling domain focus**: 100% of questions concern recycling classification (재활용) — domain-specific prompting (e.g., "이 이미지는 재활용 관련 VQA입니다") may improve model calibration.
3. **Material prediction is hardest**: "재질인 것은 무엇인가요?" has only 9.3% 4+/5 dev agreement — these are genuinely hard, visually-dependent questions.
4. **Count questions have a 2개 prior**: Predicting 2개 for all count questions gives ~39% baseline; model should visually count carefully.
5. **No slot-position bias**: No need to adjust for positional answer bias (a/b/c/d are balanced at ~25% each).
6. **Brand-name questions in test**: ~400+ test questions reference specific product brands not seen in train — model needs strong OCR / brand-recognition capability.
