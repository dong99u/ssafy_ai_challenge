LIMITATIONS
─────────────────────────────────────────────────────────────────────────────────
1. Dev set has 118 missing annotations (0.22%):
   - answer1: 2 missing (0.05%)
   - answer2: 19 missing (0.43%)
   - answer3: 34 missing (0.77%)
   - answer4: 36 missing (0.82%)
   - answer5: 27 missing (0.61%)
   
   RECOMMENDATION: Use majority voting from available annotations or exclude rows
   with >2 missing values before aggregation.

2. Answer 'b' shows systematic bias (~1.8pp above expected 25%) across all Dev
   annotators. POSSIBLE CAUSES:
   - Question formulation favoring option 'b'
   - Annotation interface default selection bias
   - Genuine semantic difficulty distribution

3. 45% question overlap across splits raises data leakage concerns:
   RECOMMENDATION: Verify no train questions appear in test set via string matching
   or embedding similarity to ensure unbiased evaluation.

4. Inter-annotator agreement (IAA) not computed:
   RECOMMENDATION: Calculate Fleiss' kappa or Krippendorff's alpha before using
   multi-annotation approach for model validation.

5. Answer option length variation (1-43 chars) may require padding:
   RECOMMENDATION: Use variable-length output embeddings or fixed-length padding
   during preprocessing.

RECOMMENDATIONS FOR MODEL DEVELOPMENT
─────────────────────────────────────────────────────────────────────────────────
1. DATA PREPARATION:
   ✓ Train/Test splits are production-ready (no preprocessing needed)
   ✓ Remove 118 rows with >1 missing Dev annotations OR use majority voting
   ✓ Verify no train/test overlap via substring/embedding similarity check

2. BASELINE METRICS:
   ✓ Chance performance (random): 25% (4-way classification)
   ✓ Majority class baseline: ~25% (balanced classes)
   ✓ Target: >70% train accuracy by epoch 5-10

3. VALIDATION STRATEGY:
   ✓ Use stratified 5-fold CV on Train (preserve 24-26% per option)
   ✓ Use Dev with majority voting for final evaluation
   ✓ Calculate class-wise F1 scores to monitor balance

4. ARCHITECTURE CONSIDERATIONS:
   ✓ Vision encoder: Use multilingual (Korean) models:
     - CLIP with Korean vocabulary
     - Qwen3-VL or other multilingual vision transformers
     - BLIP-2 multilingual variant
   
   ✓ Text encoder: Korean-optimized:
     - KoBERT, KoElectra, or Ko-RoBERTa
     - Qwen3 multilingual LLM
   
   ✓ Output: Classification head (4 classes) or multi-label attention

5. TRAINING OPTIMIZATION:
   ✓ Early stopping: Monitor Dev set majority-vote accuracy
   ✓ Class weights: Use uniform (1.0) due to balance; can add warmth=0.95 for 'b'
   ✓ Learning rate: Start 1e-4 for pretrained encoders, 1e-3 for classifier
   ✓ Augmentation: Image rotation ±15°, color jitter; no question augmentation
