
# RESEARCH STAGE 6: Image File Properties Analysis - EXECUTIVE SUMMARY

## Quick Facts
- **Total dataset:** 14,560 images (5,073 train, 5,074 test, 4,413 dev)
- **Sample analyzed:** 600 images (200 per split)
- **Format:** 100% JPEG, 100% RGB (no grayscale)
- **Mean dimensions:** ~760px width × ~945px height
- **Median dimensions:** 720px × 960px (consistent across splits)
- **Aspect ratio (median):** 0.75 (portrait orientation)
- **Mean file size:** 125KB per image
- **Resolution range:** 87% in 500-1000px bucket, 13% in 1000-2000px

## Preprocessing Recommendations

✓ **Primary recommendation:** Target 768×960 or 720×960 input resolution
  - Covers 95%+ of data without significant downsampling
  - Matches observed median and accounts for outliers

✓ **Alternative (faster):** 512×512 with aspect ratio preservation
  - ~10% quality loss on high-res images
  - Faster inference for real-time VQA

✓ **Aspect ratio strategy:** Preserve aspect ratio, pad to target size
  - Avoids crop-induced content loss
  - Handles portrait/landscape variation

✓ **Format handling:** Standard PIL/OpenCV pipeline sufficient
  - All JPEG, no conversion needed
  - All RGB, no color mode conversion

✓ **Augmentation focus:**
  - Rotation: safe up to ±15° (respects aspect ratio bounds)
  - Color jitter: standard (all RGB)
  - Avoid extreme resizing that violates 0.45-2.23 aspect ratio bounds

## Limitations

1. Sample size: 200/split (3-4% of total) - rare properties may be underrepresented
2. No semantic analysis: Image content complexity not assessed
3. No temporal analysis: Properties may vary with collection time
4. Upstream preprocessing: Evidence suggests normalization (no extreme sizes)
5. Outlier interpretation: Some outliers may be intentional (design variation, not errors)
