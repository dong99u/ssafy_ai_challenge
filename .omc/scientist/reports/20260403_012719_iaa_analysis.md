## Competition Strategy

### Accuracy Ceiling on Dev Set
If the competition evaluates using majority vote as the ground truth label:
- **Upper bound:** A model that perfectly matches majority vote = 100% dev accuracy
- **Practical ceiling:** Since 20.5% of questions have no clear majority (2-2-1 splits), a model must guess on ~20.5% of questions — maximum deterministic accuracy ≈ **79.5%** without image understanding of edge cases

### Pseudo-Labeling Recommendations

| Strategy | Samples | Label Noise Est. | Recommendation |
|----------|---------|------------------|----------------|
| Option A: 4/5+ only | 902 (20.4%) | ~0% | Highest quality, very small set |
| Option B: 3/5+ only | 3,508 (79.5%) | ~10.2% | **Recommended** — good balance |
| Option C: All (incl. no-majority) | 4,413 (100%) | ~28.0% | High noise, risky |

**Recommended approach:** Use Option B (3/5+ agreement) for pseudo-labeling. The ~10% noise rate is manageable with confidence-weighted loss. Treat no-majority questions (904) as unlabeled or use soft labels.

### Annotator Insight
Answer1 is a potential outlier annotator — consider downweighting or flagging answer1 votes when constructing consensus labels for borderline cases.

---

## [LIMITATION]
1. **No ground truth:** Difficulty and reliability are estimated from annotator agreement alone — actual visual ambiguity in images is not verified.
2. **Missing votes:** 118 vote slots are missing (2.7% of total). These were excluded per-question but may introduce slight bias.
3. **Question type classifier:** Korean keyword-based heuristic — may misclassify edge cases (e.g., counting questions that also involve color discrimination).
4. **Kappa validity:** Fleiss' Kappa assumes raters are interchangeable and ratings are independent — neither is guaranteed here if annotators saw the same images.
5. **Pseudo-label noise estimates** are conservative upper bounds derived from analytical proportions, not empirical validation.
